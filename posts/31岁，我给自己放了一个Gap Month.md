3月底，我从上一家公司离职了，说说在家休息这段时间的思考和见闻。

---

## 人生模式：React vs Plan-And-Execute

大模型 Agent 有两种常见的推理模式：React 和 Plan-And-Execute。用人话来说，就是"见机行事"和"谋定而后动"。

我之前的工作状态，就像个 React 模式的人肉 Agent。每天接收需求、工单、生产问题，thought and action; 生活也是——工作日下班后窝在出租屋里，周末再回广州，生活被切成两段。在 event loop 里转久了人就麻，自己也没有想清楚到底想要什么。

"人生规划"这词听起来虚，但落地到一条条细则上，都是具体又现实的：我想做什么样的工作、过什么样的生活；3-5年后又想做什么样的工作、过什么样的生活。我头一次花时间把这些细则一条条写下来，这大概就是 Plan-And-Execute 的 plan。

## 有欲望而无行动者滋生瘟疫

> 有欲望而无行动者滋生瘟疫——《鬼泣5》V
>
> 与其感慨路难行，不如马上出发——《Dota2》克林克兹

这两句话侧重点不同，但指向都一样：动起来。

如果你的 plan 列清楚了，就赶紧 execute。现实里，不存在一个“万事俱备”的时间点，找工作面试也一样。

```
5年以上经验的开发面试考什么？
还要背八股文吗？
还要刷 leetcode 吗？
现在是不是都问 AI 了？
```

这些问题在网上找不到标准答案，只要计划了要换工作，就直接出去面试，边面试边更新调整，就像 event loop 也要执行一轮才能 observation 和 action。

## 面试见闻

休息这一个月，面试了各种类型的公司，从 old school 国企到前沿 Web3 都接触过，风格差异挺大：

- 有的公司允许直接用 AI 编码工具，也有公司严防死守，生怕你"作弊"。
- 有面试官能根据简历内容快速理解我做过的项目，找出业务价值、重难点、可能存在的瓶颈等，进而交流架构设计和功能实现方案，自然而然地也会聊到一些八股原理；又或者是构造一个场景，考察问题分析、系统设计、AI coding工具使用能力等。也有面试官只对着简历进行关键字搜索，然后开始八股文问答，像高中英语老师检查背课文一样，他提个问题，我开始吟唱，最后来一道 leetcode 草草了事。
- 几乎所有公司都会聊 AI，但大家的理解和应用大相径庭。技术社区每天能出 100 个新的模型、协议、范式，每家企业实际落地的却各不相同，面试官的认知也各有侧重。关于 AI，大家都没标准答案——就算有了，三个月后说不定也过时了。

  我的感受是，AI 确实会重构很多岗位的职责、分工和工作方式。过去一个需求可能要严格拆成产品、前端、后端、测试、运维几段流程，未来这些边界会变得更模糊，人的价值也不一定再由某个固定工种来定义。也许以后不会再有传统意义上的“后端开发工程师”这个岗位，但互联网行业作为一种现代服务业，最终还是要服务真实业务：要有人把问题拆清楚，把工具组织起来，把方案落到业务价值上。岗位名会变，但问题不会消失。换句话说，比起守住某个 title，更重要的是成为能持续解决问题的人。

社招就是企业和候选人在互相寻找匹配的人和岗位，match 上了双方是能感觉到的，任一方觉得聊得不畅快，那就不是合适的选择。

## 输出型爱好

我离职后在家躺了不到三天就陷入了静态焦虑，俗称班瘾犯了。

最近除了准备面试，还看了一些 AI 社区的文章，想搞清现在圈子里在流行什么：Hermes、Harness Engineering、Claude Code 源码分析、Multi-Agent 等等。然后结合自己在美股买指数宽基的需求，做了一个美股信息收集和决策辅助的 Agent——[Ward](https://github.com/rainj2013/ward-agent)，也写了几篇博客记录。

另外还给朋友做了一个小说续写 Agent。朋友平时会写一些小说设定和片段，我把它做成了一个更偏创作辅助的小工具：可以读取已有的人物、世界观、情节走向，再按指定风格续写后面的内容。能用 AI Agent 解决一个真实存在的需求，哪怕需求本身不大，也还是会让人感受到自己的价值。

这段时间也认真尝试了一轮本地模型部署。之前我对本地小模型的印象还停留在“能跑但不好用”，这次重新折腾了一遍部署、推理速度、工具调用和实际任务效果。模型参数、显存占用、响应速度、上下文长度这些东西，单独看都只是数字，真拿来做 Agent，卡在哪、慢在哪、哪里容易掉链子，感受就很具体了。

人终究是需要通过输出来摆脱虚无的，可以不上狭义的班，但不能不做广义的工作。
